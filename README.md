# About Me
I am a technically focused individual with experience in infrastructure engineering, software development, and privacy‑first systems design. My work blends automation, documentation, and architectural clarity across hardware and software layers.

* Custom control interfaces in React and Go for orchestration and observability
* Monitoring, fallback logic, and automated recovery strategies for distributed and edge environments
* Self‑hosting, private‑by‑design services, and secure remote access models

## Philosophy
Infrastructure should be understandable, maintainable, and private by default. I build systems designed to withstand failure gracefully, document themselves intelligently, and operate securely outside of corporate constraints.  
Privacy and clarity aren’t optional — they are architectural pillars.

## Why Most of My Code Is Not on GitHub
GitHub is owned by Microsoft, and due to privacy concerns — specifically code analysis, training models on public repositories, and ecosystem‑wide telemetry — only a very small portion of my work is hosted here. My private repositories live on self‑hosted infrastructure where I retain full control over access, metadata, and code provenance.
